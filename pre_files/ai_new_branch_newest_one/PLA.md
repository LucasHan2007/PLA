AI 编程学习助手

1. 项目概述

1.1 项目名称

面向图像处理与计算机视觉的 AI 编程学习助手。

1.2 项目背景

图像处理、计算机视觉和深度学习项目通常包含数据读取、图像预处理、模型构建、训练优化、结果评估与可视化等多个环节。对于编程初学者而言，困难不仅在于理解单行代码，更在于理解一个完整项目应如何拆解、不同模块之间如何衔接，以及为什么需要选择特定模型、损失函数和训练策略。

传统代码问答工具通常直接给出完整代码，容易导致初学者“复制后运行”，但无法理解项目背后的设计逻辑。因此，本项目拟构建一个面向图像处理领域的 AI 编程学习助手，通过“逻辑方案—自然语言步骤—可执行代码”的分层方式，将复杂视觉项目逐步拆解，帮助学习者从任务理解、方案设计到代码实现逐步建立完整能力。

1.3 项目目标

本项目旨在帮助图像处理与计算机视觉方向的初学者降低项目开发门槛，使其能够在 AI 的引导下完成从问题分析到代码实现的全过程学习。

系统重点实现以下目标：

1. 帮助用户理解图像处理或视觉任务的整体逻辑，而不是只获得零散代码；
2. 使用苏格拉底式启发提问，引导用户主动思考任务目标、数据形式、模型选择和训练策略；
3. 将复杂视觉项目拆解为清晰、可执行的自然语言步骤；
4. 根据步骤自动生成相应代码，并提供模块级解释；
5. 对图像处理、深度学习和视觉任务中的关键术语、函数及模型结构进行补充说明；
6. 保存用户的学习过程、错误问题和项目方案，形成个人化知识库。

2. 目标用户与适用任务

2.1 目标用户

系统主要面向以下用户：

* Python 编程初学者；
* 图像处理课程学习者；
* 深度学习与计算机视觉入门者；
* 希望完成课程设计、实验项目或小型视觉应用的学生；
* 需要理解视觉模型训练流程的科研初学者。

2.2 适用任务范围

系统初期重点支持以下类型任务：

* 图像分类；
* 图像分割；
* 目标检测；
* 图像增强与去噪；
* OCR 与手写字识别；
* 人脸识别或表情识别；
* 医学图像分析；
* 缺陷检测与异常检测；
* 图像风格迁移；
* 基础视觉模型训练与部署。

例如，用户输入：

我想做一个手写数字识别项目，不知道应该从哪里开始。

系统不直接输出完整代码，而是引导用户依次明确：

1. 输入数据是什么形式，应该怎么开始；
2. 任务是分类、检测还是序列识别，识别任务应该怎么做；
3. 是否需要数据增强，数据增强的目的；
4. 使用传统机器学习还是 CNN；
5. 应该选择什么损失函数；
6. 如何训练、验证和评估模型；
7. 如何将方案拆解为可执行代码。

3. 核心设计理念

系统采用“从抽象到具体”的分层学习模式：

[
\text{项目需求}
\rightarrow
\text{逻辑方案}
\rightarrow
\text{执行步骤}
\rightarrow
\text{模块代码}
\rightarrow
\text{运行反馈与修正}
]

与直接生成代码不同，系统强调：

* 先理解任务；
* 再设计方案；
* 再拆分步骤；
* 最后生成代码；
* 根据用户运行反馈持续解释和修改。

系统中的 AI 不只是代码生成器，而应承担“项目导师”的角色。

4. 网页端整体交互设计

网页端由四个核心区域组成。

4.1 左侧：逻辑方案模块

该模块用于展示项目的宏观设计思路，包括：

* 任务目标；
* 输入与输出定义；
* 数据处理方案；
* 模型选择；
* 损失函数选择；
* 训练流程；
* 评价指标；
* 可视化与部署方案。

例如，对于“手写字识别项目”，逻辑方案可以展示为：

1. 明确任务类型：图像分类
2. 准备手写字图像与标签
3. 对图像进行灰度化、缩放和归一化
4. 构建 CNN 分类模型
5. 使用 CrossEntropyLoss 训练模型
6. 使用 Accuracy、Precision、Recall 评估结果
7. 保存模型并进行单张图像预测

该模块采用树状结构、流程图或思维导图形式呈现，让用户理解完整项目结构。

⸻

4.2 中间：自然语言执行步骤模块

该模块将逻辑方案转换为可以逐步完成的操作说明。

例如：

步骤 1：读取训练图像和对应标签。
步骤 2：将图像缩放为统一大小，例如 224×224。
步骤 3：对图像进行归一化，使像素值位于合理范围。
步骤 4：构建卷积神经网络，用于提取图像特征。
步骤 5：设置分类损失函数和优化器。
步骤 6：循环读取训练数据，计算损失并更新模型参数。
步骤 7：在验证集上测试模型准确率。
步骤 8：保存训练完成的模型。

每个步骤可展开查看：

* 为什么需要这一步；
* 所需输入是什么；
* 输出结果是什么；
* 涉及哪些 Python、PyTorch 或 OpenCV 知识；
* 对应哪一段代码。

⸻

4.3 右侧：代码显示与编辑模块

该模块根据当前自然语言步骤生成对应代码。

代码模块应支持：

* Python 代码高亮；
* 代码折叠；
* 行号显示；
* 模块化代码展示；
* 一键复制；
* 用户在线编辑；
* 代码运行结果或错误信息展示；
* 对关键行添加注释；
* 对函数、类和变量进行解释。

例如，用户点击“读取图像数据”步骤后，右侧展示：

from torchvision import datasets, transforms
transform = transforms.Compose([
    transforms.Resize((224, 224)),
    transforms.ToTensor(),
])
train_dataset = datasets.ImageFolder(
    root="data/train",
    transform=transform,
)

系统还可以标注：

* transforms.Resize：将图像缩放到统一尺寸；
* transforms.ToTensor：将图像转换为 PyTorch Tensor；
* ImageFolder：按文件夹名称自动读取类别标签。

⸻

4.4 下方：对话与追问模块

该模块提供多轮对话入口，用户可以：

* 提问某一步为什么需要；
* 请求更简单的解释；
* 请求替换模型；
* 上传报错信息；
* 上传代码片段；
* 上传图像或数据样例；
* 请求生成下一步代码；
* 请求检查当前项目结构是否合理。

系统采用苏格拉底式引导，而不是立即给出全部答案。

例如用户提出：

我想做一个手写字识别模型，应该用什么模型？

系统可以先追问：

你的输入是单个字符图像，还是一整行文字图像？
你的标签是字符类别，还是字符序列？
数据量大约有多少？
你更关注识别准确率，还是模型运行速度？

在用户回答后，系统再给出更有针对性的模型建议。

5. 核心功能模块

5.1 智能对话与项目诊断模块

该模块负责理解用户问题、识别任务类型，并构建项目方案。

主要功能包括：

* 多轮对话与上下文记忆；
* 项目需求分析；
* 任务类型识别；
* 图像处理流程推荐；
* 模型、损失函数和指标推荐；
* 苏格拉底式引导提问；
* 根据用户水平调整解释深度。

对于一个视觉项目，系统应优先分析：

[
\text{任务类型}
\rightarrow
\text{数据形式}
\rightarrow
\text{输入输出}
\rightarrow
\text{模型}
\rightarrow
\text{损失函数}
\rightarrow
\text{训练策略}
\rightarrow
\text{评价指标}
]

⸻

5.2 逻辑方案生成模块

该模块将用户自然语言需求转换为结构化项目方案。

输出内容包括：

* 项目任务定义；
* 数据流；
* 模型流程；
* 训练流程；
* 推理流程；
* 模块依赖关系；
* 风险点与常见错误。

例如：

输入：手写字图像
输出：对应字符类别
数据预处理：
灰度化 → 缩放 → 归一化 → 数据增强
模型：
CNN Backbone → Feature Extractor → Classification Head
训练：
Cross Entropy Loss → Adam Optimizer → Validation Accuracy
推理：
输入单张图像 → 模型预测 → 输出字符类别

⸻

5.3 自然语言执行步骤生成模块

该模块将抽象逻辑方案转换为具体的开发步骤。

每个步骤包含：

* 步骤编号；
* 步骤目标；
* 输入；
* 输出；
* 所需知识；
* 对应代码模块；
* 常见错误；
* 下一步建议。

例如：

步骤名称：构建数据集读取模块
目标：
读取训练图像和标签，并转换为模型可以处理的数据格式。
输入：
图像文件夹和类别标签。
输出：
PyTorch DataLoader。
关键知识：
Dataset、DataLoader、图像变换、Batch。
常见错误：
图像尺寸不一致；
标签路径错误；
训练集和验证集混合。

⸻

5.4 代码生成与代码解释模块

该模块根据当前执行步骤生成代码，并对代码进行结构化解释。

代码生成应按模块组织，例如：

project/
├── data_loader.py
├── model.py
├── train.py
├── evaluate.py
├── inference.py
└── config.py

而不是一次性生成一个超长脚本。

代码解释包括：

* 关键函数说明；
* 参数含义；
* 输入输出张量形状；
* 常见报错原因；
* 可修改参数；
* 与当前项目步骤的对应关系。

对于深度学习代码，还可自动展示：

输入图像：[B, 3, 224, 224]
经过 CNN Backbone：[B, 512]
分类输出：[B, num_classes]

5.5 学习内容增强模块

该模块用于增强用户对代码和概念的理解。

主要功能包括：

代码高亮

根据语言自动高亮 Python、JSON、YAML、Shell、SQL 等代码。

术语解释

自动识别并解释关键术语，例如：

* 缩放结果；

5.6 错误分析与调试辅助模块

用户可以粘贴报错信息、训练日志或代码片段，系统自动分析：

* 报错位置；
* 可能原因；
* 修复建议；
* 修改后的代码；
* 相关基础知识。

例如：

RuntimeError: Expected all tensors to be on the same device

系统可解释：

原因：
模型在 GPU 上，但输入图像仍在 CPU 上。
解决方法：
将输入图像和标签一起移动到 GPU。

并给出对应代码。

6. AI 输出结构设计

为了便于前端解析，建议 LLM 不直接返回自由文本，而是按照统一 JSON 格式输出。

{
  "task_summary": "构建一个手写数字分类模型",
  "logic_plan": [
    {
      "id": 1,
      "title": "任务定义",
      "content": "将输入的手写数字图像分类为 0 到 9。"
    },
    {
      "id": 2,
      "title": "数据预处理",
      "content": "将图像缩放、归一化，并构建训练集与验证集。"
    }
  ],
  "execution_steps": [
    {
      "step_id": 1,
      "title": "读取图像数据",
      "description": "读取图像文件及其标签。",
      "knowledge_points": ["Dataset", "DataLoader"],
      "code_module": "data_loader.py"
    }
  ],
  "code_blocks": [
    {
      "file_name": "data_loader.py",
      "language": "python",
      "code": "..."
    }
  ],
  "terms": [
    {
      "term": "DataLoader",
      "definition": "用于按 batch 加载训练数据。"
    }
  ],
  "follow_up_questions": [
    "你的数据集是文件夹形式还是 CSV 标注形式？"
  ]
}

这样前端可以将不同内容准确渲染到四个模块中。

11. 项目创新点

1. 面向图像处理与计算机视觉项目，而非泛化代码问答；
2. 采用“逻辑方案—执行步骤—代码实现”三层学习结构；
3. 引入苏格拉底式引导，避免用户只复制代码；
4. 将视觉任务中的数据、模型、损失、训练和评估流程显式拆解；
5. 支持代码、图像、训练日志和报错信息联合分析；
6. 通过知识沉淀模块形成个人化视觉编程学习档案；
7. 可进一步扩展为面向初学者的 AI 辅助编程研究平台。

⸻

12. 总结

该系统的核心目标不是让 AI 替用户完成全部编程，而是帮助用户理解：

[
\text{为什么这样设计}
\rightarrow
\text{应该先做什么}
\rightarrow
\text{每一步如何实现}
\rightarrow
\text{代码为什么这样写}
]

对于图像处理和计算机视觉学习者而言，系统将复杂项目拆解为可理解、可执行、可修改的学习流程，使用户逐步具备独立完成视觉项目的能力。
